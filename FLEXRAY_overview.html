Cách Node quản lý dữ liệu

Một Node trong mạng FlexRay thường là một ECU (Electronic Control Unit). Sự kết nối giữa ứng dụng (Host CPU) và mạng vật lý được thông qua FlexRay Communication Controller (CC).

1. Host CPU & CHI

CPU giao tiếp với FlexRay Controller thông qua CHI (Controller Host Interface).

  • Cấu hình mạng ban đầu.
  • Cập nhật dữ liệu cần truyền.
  • Đọc dữ liệu nhận được.

2. Message Buffers

"Kho chứa" dữ liệu trong RAM của Controller. Gán cứng với Frame ID và Hướng (TX/RX).

  • TX Buffer: Chờ đến đúng Slot thời gian để tự động bơm ra.
  • RX Buffer: Kéo data về, check CRC, báo ngắt (Interrupt) cho CPU.

3. Scheduling (Lập lịch)

Giao thức Time-Triggered. Mọi thứ được định đoạt từ trước qua Schedule Table.

  • Truyền đúng Slot dựa trên đồng hồ nội bộ đã đồng bộ.
  • Tự phát Null Frame nếu CPU quên nạp data.

Các loại Frame được truyền

1. Startup Frame

Gửi bởi Coldstart Node để đánh thức mạng và thiết lập nhịp chu kỳ ban đầu.

2. Sync Frame

Được lắng nghe để đo đạc độ lệch thời gian, từ đó đồng bộ đồng hồ nội bộ (Macrotick) cho khớp với toàn mạng.

3. Normal Data Frame

Frame chứa Dữ liệu ứng dụng thực tế, gửi ở các slot cố định (Static) hoặc theo Event (Dynamic).

4. Null Frame

Tự động phát ra khi tới lượt Slot trong Static Segment nhưng CPU chưa kịp nạp Data mới.
